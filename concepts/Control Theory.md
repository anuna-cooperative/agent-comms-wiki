# Control Theory

The engineering discipline of designing **feedback controllers** that drive a system toward a reference goal despite disturbances, and of analysing the resulting closed loop for properties such as **stability, accuracy, settling time, and overshoot** (the "SASO" properties). In self-adaptive software it is the rigorous underpinning of the [[Feedback Loop]] / [[MAPE-K]] stance: if an adaptive system is really a control loop, control theory supplies both the design methods and the *guarantees*.

The line runs from [[Feedback Control of Computing Systems]] (Hellerstein et al. 2004), which imports classical control into computing-systems performance management via system identification and controller design, through [[Engineering Self-Adaptive Systems through Feedback Loops]] (Brun et al. 2009), which argues the loop should be a first-class artefact, to [[Automated Design of Self-Adaptive Software with Control-Theoretical Formal Guarantees]] (Filieri, Hoffmann & Maggio 2014), which automates controller synthesis so software engineers get formal guarantees without doing the control-theory design by hand.
