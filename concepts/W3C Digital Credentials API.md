# W3C Digital Credentials API

A browser API, standardised at the W3C, that lets web pages request digital credentials from a user's wallet (on-device or via a remote/cross-device flow) through the user agent, building on the Credential Management mechanism. It provides the user-agent-mediated *plumbing* by which a site can invoke a credential request and receive a [[Verifiable Presentation]] — handling user consent and wallet selection — while leaving the credential format and exchange protocol (e.g. [[OpenID4VP]]) to other specs. It is one of the standards that [[x401 - HTTP Proof Requirement Protocol|x401]] composes with for its presentation-acquisition leg.
